# EXO — Instructions Copilot

## Source de vérité
`PROMPT_MAITRE.md` est le document unique. Toute question d'architecture → consulter ce fichier.

## Projet
EXO v30.0 — Assistant vocal IA offline-first (C++17 / Qt 6.9.3 / CMake / Windows).
25 microservices Python, framework cognitif hiérarchique (D:/EXO/python/).
Pipeline : capture micro → VAD → WakeWord → STT → NLU → Claude API → TTS → DSP → playback.

## Contrainte Anti-Piper
Ne JAMAIS proposer Piper TTS. Le TTS d'EXO est Orpheus 3B FR GGUF Q8 (CUDA), fallback Qt TTS.

## Stack
- **C++17**, Qt 6.9.3, CMake, MSVC 2022
- **TTS** : Orpheus 3B FR GGUF Q8 CUDA RTX 3070 (port 8767, services/orpheus/server_ws.py)
- **STT** : Whisper.cpp Vulkan AMD 6750 XT, modèle small (port 8766 → whisper-server.exe :8769)
- **LLM** : Claude `claude-sonnet-4-20250514`, SSE, 29 FC tools
- **Audio** : RtAudio WASAPI (capture 16kHz), QAudioSink (playback 16kHz)
- **Python** : 3.11 (`.venv_stt_tts` — 20 services IA/domotique) / 3.13 (`.venv` — orchestrateur + 4 services web)
- **25 services** : orchestrator :8765, stt :8766, tts :8767, vad :8768, wakeword :8770, memory :8771, nlu :8772, websearch :8773, news :8774, knowledge :8775, tools :8776, context :8777, planner :8778, executor :8779, verifier :8780, file :8781, calendar :8782, system :8783, homegraph :8784, domotic :8785, camera :8786, samsung :8787, voltalis :8788, echo :8789, network :8790
